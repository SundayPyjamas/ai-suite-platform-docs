## More than answers

A plain chat writes replies. The **Core Agent** can actually *get things done*. Here are its main powers, in plain words.

## 1. It uses tools to take action

Tools let the agent do real things beyond writing — like searching, looking something up, or working with a service you've connected.

- Ask *"Find the latest info on X and summarize it"* and it can search, then write.
- Connect your tools in [Integrations](/user-guide/integrations), and the agent can use them.

<Frame caption="The agent using a tool as part of answering your request.">
  <img src="/images/user-guide/agents-capabilities-tools.png" alt="An agent using a tool while working on a task" />
</Frame>

## 2. It takes multiple steps on its own

For a bigger request, the agent breaks the work into steps and does them in order — so you don't have to guide every move.

> You: *"Gather this week's sign-ups, group them by country, and write a short summary."*
> The agent: gathers → groups → writes — then shows you the result.

You can watch these steps as it works, which makes it easy to trust. See [Run & share](/user-guide/agents/run-and-share).

## 3. It creates files and artifacts

This is a big one. The Core Agent can produce **actual files** — a document, a spreadsheet-style output, code, or a written report — not just text in the chat.

<Steps>
<Step title="Ask for something concrete">
  *"Write this up as a document I can download."*
</Step>
<Step title="It appears in the preview panel">
  The file opens in the panel on the right, where you can read it in full.

  <Frame caption="A file the agent created, open in the preview panel.">
    <img src="/images/user-guide/agents-capabilities-artifact.png" alt="A generated file shown in the preview panel next to the chat" />
  </Frame>
</Step>
<Step title="Copy or download it">
  Use the copy or download buttons to take the file with you.
</Step>
</Steps>

<Tip>
"Artifact" is just a fancy word for **a thing the agent made** — usually a file. If you see that word, think "the finished document."
</Tip>

## 4. It can run sub-tasks

For complex jobs, the agent can spin off smaller helper tasks (sometimes called **subagents**) and bring the results back together. You don't manage these — the agent handles it. You'll just see it working through the pieces.

## 5. It remembers your sessions

Every conversation is saved as a **session**, so the agent keeps the context of what you were doing. Reopen a session later and continue as if you never left. See [Using the Agents tab](/user-guide/agents/using-the-agents-tab).

## Putting it together — an example

> **You:** *"Look up our top 5 selling products this month, then write a one-page summary I can download, with a short recommendation for each."*

The Core Agent will: use tools to gather the data → work through it step by step → write the summary → drop a downloadable file in the preview panel. One request, a finished document.
