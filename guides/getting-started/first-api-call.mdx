## Prerequisites

Before making your first API call, ensure you have:

- An active SundayPyjamas account
- A valid API key (generated from your workspace settings)
- Basic understanding of HTTP requests

## Your First API Call

The simplest way to get started is with a basic chat completion request. Here's what you need to know:

### Request Structure

Every API request requires:
- **Endpoint URL**: `https://suite.sundaypyjamas.com/api/v1/chat`
- **Authorization**: Bearer token with your API key
- **Content-Type**: `application/json`
- **Request Body**: JSON with your messages

### Example Request

<CodeGroup>
```javascript JavaScript (Node.js)
const response = await fetch('https://suite.sundaypyjamas.com/api/v1/chat', {
  method: 'POST',
  headers: {
    'Authorization': 'Bearer spj_ai_your_api_key_here',
    'Content-Type': 'application/json',
  },
  body: JSON.stringify({
    messages: [
      {
        role: 'user',
        content: 'Hello! Introduce yourself and explain what you can help me with.'
      }
    ]
  })
});

const data = await response.json();
console.log(data);
```

```python Python
import requests

response = requests.post(
    'https://suite.sundaypyjamas.com/api/v1/chat',
    headers={
        'Authorization': 'Bearer spj_ai_your_api_key_here',
        'Content-Type': 'application/json',
    },
    json={
        'messages': [
            {
                'role': 'user',
                'content': 'Hello! Introduce yourself and explain what you can help me with.'
            }
        ]
    }
)

print(response.json())
```

```bash cURL
curl -X POST https://suite.sundaypyjamas.com/api/v1/chat \
  -H "Authorization: Bearer spj_ai_your_api_key_here" \
  -H "Content-Type: application/json" \
  -d '{
    "messages": [
      {
        "role": "user",
        "content": "Hello! Introduce yourself and explain what you can help me with."
      }
    ]
  }'
```
</CodeGroup>

## Understanding the Response

The API returns a structured response with the following format:

```json
{
  "id": "chat_abc123",
  "object": "chat.completion",
  "created": 1677832384,
  "model": "llama-3.3-70b-versatile",
  "choices": [
    {
      "index": 0,
      "message": {
        "role": "assistant",
        "content": "Hello! I'm the SundayPyjamas AI assistant, powered by advanced language models. I can help you with:\n\n• Content generation and writing assistance\n• Code development and debugging\n• Creative writing and brainstorming\n• Research and analysis\n• Educational explanations\n• And much more!"
      },
      "finish_reason": "stop"
    }
  ],
  "usage": {
    "prompt_tokens": 25,
    "completion_tokens": 67,
    "total_tokens": 92
  }
}
```

### Response Fields Explained

- **`id`**: Unique identifier for the chat completion
- **`object`**: The object type (`chat.completion`)
- **`created`**: Unix timestamp of when the request was made
- **`model`**: The AI model used for the completion
- **`choices`**: Array of completion choices (usually just one)
- **`usage`**: Token usage information for billing and optimization

## Testing Your First Request

<Steps>
<Step title="1. Prepare Your Environment">

Choose your preferred method:
- **Browser Console**: For quick testing
- **Terminal**: With curl for command-line testing
- **Code Editor**: For Node.js or Python scripts
</Step>

<Step title="2. Replace the API Key">

Make sure to replace `spj_ai_your_api_key_here` with your actual API key from your workspace settings.

<Warning>
Never share your API key publicly or commit it to version control.
</Warning>
</Step>

<Step title="3. Execute the Request">

Run your code or command and observe the response. You should receive a JSON response similar to the example above.
</Step>

<Step title="4. Handle the Response">

Extract the assistant's message from the response:

```javascript
const data = await response.json();
const message = data.choices[0].message.content;
console.log(message);
```
</Step>
</Steps>

## Common Issues and Solutions

### Authentication Errors

**Error**: `401 Unauthorized`
- **Solution**: Check your API key format and ensure it's active
- **Solution**: Verify the Authorization header format: `Bearer spj_ai_your_api_key_here`

### Request Errors

**Error**: `400 Bad Request`
- **Solution**: Validate your JSON structure
- **Solution**: Ensure `messages` is an array with valid role/content pairs

### Network Issues

**Error**: Connection timeout or network errors
- **Solution**: Check your internet connection
- **Solution**: Verify the API endpoint URL
- **Solution**: Ensure HTTPS is used (not HTTP)
