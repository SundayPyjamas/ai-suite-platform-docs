## Creating Your Account

### Sign Up Process

<Steps>
<Step title="1. Visit the Platform">
Navigate to [SundayPyjamas AI Suite](https://suite.sundaypyjamas.com) and click **"Sign Up"** in the top right corner.
</Step>

<Step title="2. Choose Your Authentication Method">
Select your preferred sign-in method:
- **Email & Password**: Create a new account
- **Google OAuth**: Quick sign-in with Google
- **GitHub OAuth**: Sign-in with your GitHub account
- **SSO**: For enterprise users with custom domain
</Step>

<Step title="3. Verify Your Email">
Check your email for a verification link and click it to activate your account.
</Step>

<Step title="4. Complete Your Profile">
Add basic information to personalize your experience:
- Full name
- Organization (optional)
- Preferred timezone
- Communication preferences
</Step>
</Steps>

## Setting Up Your Workspace

### Workspace Creation

Once your account is created, you'll be prompted to create your first workspace:

<Steps>
<Step title="1. Choose Workspace Type">
Select the appropriate workspace type for your use case:
- **Personal**: For individual developers and small projects
- **Team**: For collaborative development with multiple users
- **Enterprise**: For organizations with advanced security and compliance needs
</Step>

<Step title="2. Configure Workspace Settings">
Set up your workspace with these essential configurations:
- **Workspace Name**: Choose a descriptive name
- **URL Slug**: Unique identifier for your workspace
- **Region**: Select the geographic region for data residency
- **Team Members**: Add initial team members (for Team/Enterprise plans)
</Step>

<Step title="3. Review Terms & Agreements">
Accept the platform terms of service and privacy policy to continue.
</Step>
</Steps>

## API Key Generation

### Generating Your First API Key

<Steps>
<Step title="1. Navigate to API Settings">
From your workspace dashboard, go to **Settings** → **API** → **API Keys**.
</Step>

<Step title="2. Create New API Key">
Click **"Generate API Key"** and configure:
- **Key Name**: Descriptive name (e.g., "Development", "Production")
- **Environment**: Development, Staging, or Production
- **Permissions**: Select appropriate access levels
- **Rate Limits**: Custom rate limits (if available in your plan)
</Step>

<Step title="3. Secure Your API Key">
Copy and store your API key immediately - it won't be shown again:
```bash
spj_ai_a1b2c3d4e5f6789012345678901234567890abcdef123456789012345678901234
```
<Warning>
**Security Critical**: Never share your API key publicly, commit it to version control, or expose it in client-side code.
</Warning>
</Step>

<Step title="4. Configure Environment Variables">
Store your API key securely:
```bash
# .env file
SUNDAYPYJAMAS_API_KEY=spj_ai_a1b2c3d4e5f6789012345678901234567890abcdef123456789012345678901234
```
</Step>
</Steps>

## Testing Your Setup

### Verify API Key
Test your API key with a simple request:

<CodeGroup>
```bash cURL Test
curl -X POST https://suite.sundaypyjamas.com/api/v1/chat \
  -H "Authorization: Bearer $SUNDAYPYJAMAS_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"messages": [{"role": "user", "content": "Hello!"}]}'
```

```javascript JavaScript Test
const testApiKey = async () => {
  const response = await fetch('https://suite.sundaypyjamas.com/api/v1/chat', {
    method: 'POST',
    headers: {
      'Authorization': `Bearer ${process.env.SUNDAYPYJAMAS_API_KEY}`,
      'Content-Type': 'application/json',
    },
    body: JSON.stringify({
      messages: [{ role: 'user', content: 'Hello!' }]
    })
  });

  if (response.ok) {
    console.log('✅ API key is working correctly');
    return await response.json();
  } else {
    console.error('❌ API key test failed:', response.status, response.statusText);
  }
};
```
</CodeGroup>

## Best Practices

### API Key Management
- **Rotate regularly**: Generate new keys every 90 days
- **Use environment variables**: Never hardcode keys in your code
- **Separate keys**: Use different keys for different environments
- **Monitor usage**: Set up alerts for suspicious activity

### Workspace Organization
- **Use descriptive names**: Make it clear what each workspace is for
- **Set up proper permissions**: Follow principle of least privilege
- **Enable audit logging**: Track all API usage and changes
- **Regular backups**: Export important configurations

### Security Checklist
- [ ] Enable 2FA for all users
- [ ] Set up IP restrictions if needed
- [ ] Configure billing alerts
- [ ] Enable audit logging
- [ ] Set up usage monitoring
- [ ] Review team permissions regularly

## Troubleshooting

### Common Setup Issues

<AccordionGroup>
<Accordion title="Email Not Received">
- Check your spam folder
- Verify your email address is correct
- Contact support if issue persists
</Accordion>

<Accordion title="API Key Generation Failed">
- Verify your account has proper permissions
- Check if you've reached the key limit for your plan
- Ensure your workspace is active
</Accordion>

<Accordion title="Workspace Creation Error">
- Choose a unique workspace URL slug
- Verify your account is fully verified
- Check your plan limits for workspace count
</Accordion>
</AccordionGroup>
